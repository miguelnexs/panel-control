I will implement a comprehensive configuration module for managing the Google API Key, specifically designed for Gmail API integration. This involves backend updates to securely store and validate the key, and a frontend interface for user interaction.

### Backend Implementation
1.  **Modify `AppSettings` Model (`backend/config/models.py`)**:
    *   Add a `google_config` field (`JSONField`) to store the Google API configuration.
    *   This field will store the API key and potentially other settings (e.g., sender email).

2.  **Update `AppSettingsSerializer` (`backend/webconfig/api.py`)**:
    *   Include `google_config` in the serializer fields.
    *   Implement `validate_google_config` to encrypt the `api_key` before saving using the existing `encrypt_text` utility.
    *   Ensure the `update` method preserves the existing encrypted key if a new one isn't provided (handling the masked display value from frontend).

3.  **Create Google Connection Test Endpoint**:
    *   Create a new view `TestGoogleConfigView` in `backend/webconfig/api.py`.
    *   This view will accept an API key, attempt to connect to a Google API endpoint (e.g., `https://gmail.googleapis.com/discovery/v1/apis/gmail/v1/rest` or a simple validation endpoint using the key), and return success or specific error messages.
    *   Register the new endpoint in `backend/webconfig/urls.py`.

### Frontend Implementation
1.  **Create `GoogleConfig` Component (`frontend/src/renderer/src/components/GoogleConfig.tsx`)**:
    *   Develop a React component similar to `AlegraConfig`.
    *   **Form Fields**:
        *   `api_key`: Password input field (masked).
        *   `sender_email`: Email input field (optional, but good for context).
    *   **Actions**:
        *   "Probar Conexión" (Test Connection): Calls the new backend test endpoint.
        *   "Guardar Configuración" (Save): Saves the encrypted key to `AppSettings`.
    *   **Feedback**: Show success/error messages based on test results and save status.

2.  **Integrate into `ConfigPage` (`frontend/src/renderer/src/components/ConfigPage.tsx`)**:
    *   Add a new tab "Google" to the configuration page.
    *   Render the `GoogleConfig` component when this tab is active.
